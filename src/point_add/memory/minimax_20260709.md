## Alternatives Per Subroutine

### Alt 1: Bogan 2-CCX cswap
- **Construction**: CCZ + 2×(CX + CCX) = 2 CCX effective (Bogan et al. 2024)
- **T saving**: ~1 CCX/swap × 1032 swaps = ~44K CCX
- **Q change**: 0 (same qubit count)
- **Score impact**: 1,276K × 1,152 = **1,470M (+3.4%)**
- **Risk**: MEDIUM — dirty ancilla reuse, must verify correct at all 9024 test points
- **Reference**: `https://arxiv.org/abs/2404.09516`

### Alt 2: cswap width-truncate (TRIED — FAILED)
- **Attempt**: truncate reverse-pass cswap to `current_n = SCHED_J2[i]`
- **Result**: FAIL 9024/9024 classical mismatches
- **Root cause**: binary GCD requires symmetrical full-register swaps in reverse pass
- **Status**: NOT AVAILABLE — reverse cswap must use full n=256

### Alt 3: pseudo-Mersenne controlled_add/sub (H1 — FAILED)
- **Construction**: Algorithm 10 from Qarton's pseudo-Mersenne arithmetic
- **Result**: FAIL correctness (9024 mismatches)
- **Root cause**: cout qubit aliasing in overflow correction step
- **Status**: REVERTED — not viable without deeper analysis

### Alt 4: Square-add fusion (peak Q reduction)
- **Idea**: fuse mod_square with adjacent field add, share workspace qubits
- **Q saving**: ~256 qubits (4 registers × 256 bits → 3 registers × 256 bits)
- **Score impact**: 1,320K × 896 = **1,183M (+22%)**
- **Risk**: MEDIUM-HIGH — correctness of fused operation must be verified
- **Key insight**: cuccaro arithmetic needs O(n) extra workspace per operation — cannot be width-truncated

### Alt 5: JUMP=3 dialog codec
- **Idea**: each dialog symbol encodes 3 GCD iterations instead of 2
- **Dialog overhead reduction**: ~33% (516 → 344 symbols forward+reverse)
- **Score impact**: ~5–15% T reduction (dialog overhead portion)
- **Risk**: HIGH — codec must be exactly inverting (bijective), complex
- **Plan**: implement `DialogCodec::Triple3` with Gray-code compression

### Alt 6: RNS arithmetic (Chevignard-Fouque-Schrottenloher 2026)
- **Idea**: use RNS representation for field ops, avoid full modular inverse
- **T change**: O(n⁴log²n) increase for RNS multiply
- **Q change**: significant Q reduction possible
- **Reference**: `https://eprint.iacr.org/2026/280`

### Alt 7: Karatsuba for f=0x1001 (H2 — planned)
- **Idea**: use Karatsuba O(n^1.58) for field multiplication with small constant f
- **Target**: reduce mod_square T-count
- **Status**: PLANNED — not yet researched

## Key Findings

1. **cuccaro arithmetic cannot be width-truncated**: needs O(n) extra workspace regardless of input width
2. **reverse cswap MUST use full n=256**: binary GCD symmetry requirement
3. **forward cswap ALREADY uses current_n**: no further truncation possible there
4. **cswap is the DOMINANT T×Q contributor** (660K), followed by mod_square (4.5M)
5. **Q=1152 is the binding constraint**: peak reduction gives better score improvement than T reduction
